# Health Check Endpoint Specification

> **Version:** 1.0
> **Audience:** Application developers and harness agents
> **Schema:** [`harness_skills/schemas/health_check_response.schema.json`](../harness_skills/schemas/health_check_response.schema.json)

---

## Overview

Every application managed by the harness **must** expose a health check endpoint.  Agents poll this endpoint — via a [`HealthCheckSpec`](#healthcheckspec-agent-polling-contract) generated by `harness_skills.boot.generate_health_check_spec()` — to verify an isolated instance is alive and ready before sending test traffic.

```
GET /health          (default path; configurable via BootConfig.health_path)
HEAD /health         (lightweight probe — omits the body)
```

The endpoint must:

- Return **HTTP 200** when the instance is healthy or degraded.
- Return **HTTP 503** when the instance is unhealthy.
- Respond within **5 seconds** (the harness per-request timeout default).
- Always return **`Content-Type: application/json`** for GET requests.

---

## Response Body

Defined by the JSON Schema at `harness_skills/schemas/health_check_response.schema.json`.  The minimal required body is:

```json
{
  "status": "healthy",
  "timestamp": "2026-03-20T14:32:01Z"
}
```

All fields:

| Field | Type | Required | Description |
|---|---|---|---|
| `status` | `"healthy" \| "degraded" \| "unhealthy"` | **yes** | Coarse verdict. See [Status semantics](#status-semantics). |
| `timestamp` | ISO-8601 string | **yes** | UTC time the response was generated. |
| `version` | string \| null | no | App version (semver, git SHA, or build tag). |
| `uptime_s` | number \| null | no | Seconds the process has been running. |
| `checks` | `CheckResult[]` | no | Per-subsystem check results. |
| `instance` | `InstanceInfo` | no | Harness instance correlation metadata. |

### `CheckResult`

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | **yes** | Subsystem identifier (`"database"`, `"redis"`, `"migrations"`, …). |
| `status` | `"pass" \| "fail" \| "warn"` | **yes** | Subsystem verdict. |
| `latency_ms` | integer \| null | no | Round-trip probe latency. |
| `message` | string \| null | no | Human/agent-readable detail or error message. |
| `error_code` | string \| null | no | Machine-readable code for automated remediation routing. |

### `InstanceInfo`

Populated automatically by harness-aware applications using environment variables injected at boot:

| Field | Type | Env var source | Description |
|---|---|---|---|
| `worktree_id` | string \| null | `HARNESS_WORKTREE_ID` | Matches `BootConfig.worktree_id`. |
| `port` | integer \| null | `PORT` | Matches `IsolationConfig.port`. |
| `pid` | integer \| null | — | OS process ID (`os.getpid()`). |
| `git_sha` | string \| null | `GIT_SHA` | Commit SHA deployed. |
| `git_branch` | string \| null | `GIT_BRANCH` | Branch at build time. |
| `db_schema` | string \| null | `DB_SCHEMA` / `DATABASE_URL` | Active isolated DB schema or file. |
| `environment` | string \| null | `APP_ENV` / `ENVIRONMENT` | Runtime environment name. |

---

## Status Semantics

| `status` | HTTP code | Agent behaviour |
|---|---|---|
| `"healthy"` | 200 | Boot confirmed. Proceed with test traffic. |
| `"degraded"` | 200 | Boot confirmed, but at least one non-critical check is `"warn"`. Agents may proceed with caution; log the degraded checks. |
| `"unhealthy"` | 503 | Do **not** send traffic. Inspect `checks[].error_code`, attempt a re-boot, or report failure to the state service. |

Agents determine degraded vs unhealthy by examining `checks[].status`:

- Any `"fail"` → top-level `status` must be `"unhealthy"`.
- Any `"warn"` and no `"fail"` → top-level `status` must be `"degraded"`.
- All `"pass"` → top-level `status` must be `"healthy"`.

## `HealthCheckSpec` — Agent Polling Contract

Agents never hard-code health URLs.  They receive a `HealthCheckSpec` generated by `harness_skills.boot.generate_health_check_spec()` and poll using `harness_skills.boot._poll_health_check()`:

```python
from harness_skills.boot import BootConfig, IsolationConfig, generate_health_check_spec

config = BootConfig(
    worktree_id="fb563322",
    start_command="uvicorn myapp.main:app --port 8001",
    isolation=IsolationConfig(port=8001),
    health_path="/health",          # must match the endpoint path above
    health_timeout_s=30.0,
    health_interval_s=1.0,
)

spec = generate_health_check_spec(config)
# spec.url          → "http://localhost:8001/health"
# spec.method       → HealthCheckMethod.GET
# spec.expected_codes → [200, 201, ..., 299]
# spec.timeout_s    → 5.0
# spec.interval_s   → 1.0
# spec.max_wait_s   → 30.0
```

The spec JSON representation (for cross-process hand-off):

```json
{
  "url": "http://localhost:8001/health",
  "method": "GET",
  "expected_codes": [200, 201, 202, 203, 204, 205, 206, 207, 208, 226],
  "timeout_s": 5.0,
  "interval_s": 1.0,
  "max_wait_s": 30.0,
  "headers": {}
}
```

### Polling algorithm (pseudo-code)

```
start_time = now()
while elapsed < spec.max_wait_s:
    response = HTTP(spec.method, spec.url,
                    timeout=spec.timeout_s, headers=spec.headers)
    if response.status_code in spec.expected_codes:
        return READY
    sleep(spec.interval_s)
return TIMED_OUT
```

> **Note:** Agents must treat both `"healthy"` and `"degraded"` status values as boot-confirmed (HTTP 200 is the signal, not the JSON body status).  The JSON body is inspected only after boot is confirmed, to decide whether to log warnings or skip certain checks.

## Minimal FastAPI Implementation

For projects using FastAPI + `anyio` (already in the dependency tree):

```python
# myapp/routes/health.py
from __future__ import annotations

import os
import time
from datetime import datetime, timezone

from fastapi import APIRouter
from fastapi.responses import JSONResponse

router = APIRouter()

_START_TIME = time.monotonic()


@router.get("/health", tags=["ops"])
@router.head("/health", tags=["ops"])
async def health() -> JSONResponse:
    checks = []
    overall = "healthy"

    # --- database check (example) ---
    try:
        from myapp.db import engine  # noqa: PLC0415
        async with engine.connect() as conn:
            t0 = time.monotonic()
            await conn.execute("SELECT 1")
            latency_ms = int((time.monotonic() - t0) * 1000)
        checks.append({"name": "database", "status": "pass",
                        "latency_ms": latency_ms, "message": None,
                        "error_code": None})
    except Exception as exc:  # noqa: BLE001
        checks.append({"name": "database", "status": "fail",
                        "latency_ms": None,
                        "message": str(exc),
                        "error_code": "DB_CONNECTION_REFUSED"})
        overall = "unhealthy"

    body = {
        "status": overall,
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "version": os.getenv("GIT_SHA"),
        "uptime_s": round(time.monotonic() - _START_TIME, 1),
        "checks": checks,
        "instance": {
            "worktree_id": os.getenv("HARNESS_WORKTREE_ID"),
            "port": int(os.getenv("PORT", "8000")),
            "pid": os.getpid(),
            "git_sha": os.getenv("GIT_SHA"),
            "git_branch": os.getenv("GIT_BRANCH"),
            "db_schema": os.getenv("DB_SCHEMA") or os.getenv("DATABASE_URL"),
            "environment": os.getenv("APP_ENV", "unknown"),
        },
    }
    http_status = 200 if overall in ("healthy", "degraded") else 503
    return JSONResponse(content=body, status_code=http_status)
```

Register in your app factory:

```python
from myapp.routes.health import router as health_router
app.include_router(health_router)
```

## Minimum Viable Flask Implementation

```python
# myapp/routes/health.py
import os, time
from datetime import datetime, timezone
from flask import Blueprint, jsonify

health_bp = Blueprint("health", __name__)
_START = time.monotonic()

@health_bp.get("/health")
def health():
    body = {
        "status": "healthy",
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "version": os.getenv("GIT_SHA"),
        "uptime_s": round(time.monotonic() - _START, 1),
        "checks": [],
        "instance": {
            "worktree_id": os.getenv("HARNESS_WORKTREE_ID"),
            "port": int(os.getenv("PORT", "5000")),

            "db_schema": os.getenv("DB_SCHEMA"),
            "environment": os.getenv("APP_ENV", "unknown"),
        },
    }
    return jsonify(body), 200
```

## Standard `error_code` Values

Agents use `error_code` to route automated remediation.  Use these codes consistently across applications:

| `error_code` | Meaning | Suggested agent action |
|---|---|---|
| `DB_CONNECTION_REFUSED` | Database TCP connection rejected | Check `DATABASE_URL`; verify DB container is up |
| `DB_AUTH_FAILED` | Database authentication error | Check credentials in env vars |
| `MIGRATION_PENDING` | Unapplied migrations detected | Run migration command; re-boot |
| `MIGRATION_FAILED` | Migration apply error | Inspect migration logs; escalate |
| `REDIS_CONNECTION_REFUSED` | Redis TCP connection rejected | Verify Redis is running |
| `REDIS_HIGH_LATENCY` | Redis latency above threshold | Advisory only; proceed with caution |
| `DISK_FULL` | Disk usage above 95% | Free disk space; abort if critical path |
| `EXTERNAL_API_UNREACHABLE` | Third-party API not responding | Skip integration tests; mark as degraded |
| `PORT_CONFLICT` | Bound port already in use | Re-assign port in `IsolationConfig` |

## Checklist for Application Authors

- [ ] `GET /health` returns HTTP **200** when healthy or degraded
- [ ] `GET /health` returns HTTP **503** when unhealthy
- [ ] `HEAD /health` is supported (same codes, no body)
- [ ] Response body validates against `health_check_response.schema.json`
- [ ] `instance.worktree_id` is populated from `$HARNESS_WORKTREE_ID`
- [ ] `instance.port` is populated from `$PORT`
- [ ] `instance.environment` is **never** `"production"` in test environments
- [ ] Each critical subsystem has a named `CheckResult` entry
- [ ] `CheckResult.error_code` uses values from the [standard table](#standard-error_code-values)
- [ ] Endpoint responds within **5 seconds** under load
